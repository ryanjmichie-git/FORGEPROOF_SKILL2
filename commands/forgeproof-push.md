# ForgeProof Push — Create PR with Provenance

You are creating a GitHub PR from a ForgeProof run.

---

## Steps

1. **Read run state:** Read `.forgeproof/last-run.json`. If it doesn't exist, say: "No ForgeProof run found. Run `/forgeproof <issue-number>` first."

2. **Extract metadata** from last-run.json:
   - `issue_number`, `issue_title`, `repo`, `branch`, `files_changed`, `requirements_met`, `requirements_total`, `tests_passed`, `tests_total`, `rpack_path`

3. **Create and switch to branch:**
   ```bash
   git checkout -b forgeproof/issue-<issue_number>
   ```
   If the branch already exists, switch to it: `git checkout forgeproof/issue-<issue_number>`

4. **Stage files:**
   ```bash
   git add <each file in files_changed>
   git add <rpack_path>
   ```
   Do NOT stage `.forgeproof/lib/`, `.forgeproof/ephemeral.*`, `.forgeproof/last-run.json`, or `.forgeproof/decision-log.jsonl`.

5. **Commit:**
   ```bash
   git commit -m "feat: implement issue #<issue_number> — <issue_title>

   Generated by ForgeProof with signed provenance bundle.
   Requirements: <requirements_met>/<requirements_total> met
   Tests: <tests_passed>/<tests_total> passed"
   ```

6. **Push:**
   ```bash
   git push -u origin forgeproof/issue-<issue_number>
   ```

7. **Create PR:**
   ```bash
   gh pr create \
     --title "feat: <issue_title>" \
     --body "## Summary

   Automated implementation for issue #<issue_number>: <issue_title>

   ## Changes

   <bulleted list of files_changed>

   ## Requirements Coverage

   <requirements_met>/<requirements_total> requirements met

   ## Test Results

   - Tests: <tests_passed>/<tests_total> passed

   ## Provenance

   This PR includes a cryptographically signed provenance bundle verifying the AI-generated code.

   - **Bundle:** \`<rpack_path>\`
   - **Signing:** Ed25519 (ephemeral keypair)
   - **Verify:** \`/forgeproof-verify <rpack_path>\`

   ---
   Generated by [ForgeProof](https://github.com/ryanjmichie-git/ForgeProof)"
   ```

8. **Report:** Print the PR URL and a summary.

9. **Switch back:** `git checkout -` to return to the previous branch.
